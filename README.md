## ğŸ¤– Connect Your AI Agent

**Point your OpenClaw agent to `SKILL.md`** in this repo.

The skill teaches your agent how to:
- Send heartbeats (show up as "active")
- Log activities (create audit trail)
- Update tasks (move through pipeline)
- Track metrics

### For OpenClaw Users

Copy `SKILL.md` to your agent's skills folder, or reference it directly:

```
Your agent can now use Mission Control.
See SKILL.md for the full API reference.
```

### Manual Integration

Your agent just needs to make HTTP calls:

```bash
# Agent wakes up
curl -X POST "https://your-dashboard.vercel.app/api/heartbeat" \
  -H "Content-Type: application/json" \
  -d '{"name": "Forge", "status": "active"}'

# Agent does something
curl -X POST "https://your-dashboard.vercel.app/api/activity" \
  -H "Content-Type: application/json" \
  -d '{"agent": "Forge", "action": "Built new feature"}'

# Agent updates a task
curl -X POST "https://your-dashboard.vercel.app/api/task-update" \
  -H "Content-Type: application/json" \
  -d '{"title": "Build feature X", "status": "done"}'
```

## ğŸ“¡ API Reference

| Endpoint | Method | Body | Description |
|----------|--------|------|-------------|
| `/api/heartbeat` | POST | `{name, status}` | Update agent status |
| `/api/activity` | POST | `{agent, action}` | Log an activity |
| `/api/task-create` | POST | `{title, description, status, assignee, priority}` | Create task |
| `/api/task-update` | POST | `{title, status}` | Update task status |
| `/api/metric` | POST | `{name, value, agent}` | Track a metric |

**Status options:** `active`, `idle`, `sleeping`, `error`

**Task status:** `inbox`, `in_progress`, `review`, `done`, `blocked`

âš ï¸ Use `in_progress` (underscore), NOT `in-progress`.

- Verify agent name matches exactly

**Tasks not updating?**
- Use `in_progress` (underscore)

- Run `npx convex dev` in a separate terminal
